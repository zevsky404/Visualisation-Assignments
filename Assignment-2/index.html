Assignment 2

Introduction

It is time to show case your Javascript skills with d3.js! In this assignment you will apply your knowledge from the previous assignment about visual encoding. You will also be required to implement, using d3.js, two well-known multi-attributes visualization techniques, namely the scatter plot matrix and the parallel coordinates which have been introduted in the lecture. Brushing technique will be used to interact with the data. Optionally, you can also implement linking to show the brushed data simultaneously on both plots. Overall, your tasks are:

  1. Load and filter the dataset
  2. Decide on 3 numerical attributes and 1 categorical attributes from the given dataset you would to display!
  3. Create the attribute labels.
  4. Create the scatter plot matrix of those 4 attributes with the brushing technique
  5. Create the parallel coordinates plot of those 4 attributes with the brushing technique
  6. Optional, link both plots with the brush together
For this exercise, all your solution must be implemented by yourself in index.js script. Your solution will appear under div#visualization element in the index.html.

Task 1: Prepare your data (10%)

To start, you have to prepare your data and label before implementing the visualization technique. In the global scope of the index.js you are tasked with:

  1. Loading and filtering the dataset.
  2. Defining your 3 numerical and 1 categorical attributes
  3. Defining the color mapping/scheme you are using for labeling your data points and lines in both plots
Hint: Take a quick look into the dataset and decide what kind of information that you want to convey with both the Scatter Plot Matrix and the Parallel Coordinates! Filtering the dataset is useful to avoid cluttering and elminate irrelevant data. Regarding coloration, take a look at d3.scaleOrdinal.

Task 2: Create colored label (10%)

Before visualizing the dataset in each plots, it is often beneficial to encode the attributes and show first the labels to convey identity or magnitude of the data. In this function you need to make a consistent color labelling, so that viewers can associate which data to which color within and across multiple plots. Implement your label in createLabel() function!

Hints: Use the color mapping/scheme you defined before to serve as the visual channel for the label. You are free to decide which marks you are using for this. Depending on the attribute you want to encode here, text label might be useful.

Task 3: Create the scatter plot matrix for each numerical attributes pair (35%)

In a scatter plot matrix, each attribute is assigned to one column and one row of a matrix such that each cell contains a combination of two attributes that are then depicted by a scatter plot. A helper function createScatterPlotMatrix function is provided to create 3 x 3 matrix space for scatter plots for each pair of numerical attributes. You are required to express the pairwise combination attribute value of the data on each cell. In this task, you need to:

  1. Show the points for all the data accordingly
  2. Create the axes with sufficient ticks
Write your implementation in the scatterPlot function! You are free to take ideas from web examples, but we require you to write your own code.

Hints: You might encounter some data that are not properly read/loaded due to mistakes in the dataset itself. For this, you might need to use parseFloat so that all numerical attribute values is guaranteed to be stored as floats. All the data exist in the different cells so make sure to scale the attributes value according the axes and transform their position to the correct place!

Task 4: Create horizontal parallel coordinates (35%)

Realize the parallel coordinate plot with the same attributes as the scatter plot matrix. In contrast to scatter plot matrixes, in a parallel coordinates plot each attribute is represented by a single axis parallel to all others and each data item shows as a polyline instead of a point. You are expected to

  1. Show the vertical axes next after the other horizontally
  2. Label the axes and show the ticks
  3. Produces connected lines for all data
Use the same color scheme as the previous task.

Hints: To implement the parallel coordinates plot, you will need a storage (for example a map) to hold your scales for each axis. The usage of an JavaScript object is not recommended here, since it is not iterable and, therefore, cannot be used to create the axes with the usual join pattern. Another useful tool, provided by d3 is the d3.line function, that -- given an array of [x,y] values -- creates a string for an SVG (cor canvas) path, that can then be used inside a SVG path element's d attribute. Same as previous task, if you encounter problem with the data value due to conversion problem parseFloat might help.

Task 5: Brushing (10%)

Add interactivity for both scatter plot matrix and parallel coordinate plot with brushing, such that selected data is shown much more clearly than the rest. Write your implementation in brushed function.

Hints: The example of a brush within a single scatter plot has already given you some hints as to how a d3.brush can be used. Other examples in various contexts can be found here. Please note, that there is not only d3.brush , but also d3.brushX or d3.brushY which might make more sense in the context of the parallel coordinates plot.

Task 6: Linking (Optional 10%)

To benefit more from both views and better understand how a scatter plot would translate into parallel coordinates and the other way around, linking selections of both visualizations is essential. Here you are to connect the scatter plots within the scatter plot matrix and the parallel coordinates and modify the brushed function, such that selected data in the PCP will be reflected onto the all the plots of the scatter plot matrix and vise versa.

Hint: To implement the linking you have to make use of the selectedItems variable.
