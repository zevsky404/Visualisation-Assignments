Treemaps

Previously, we introduced some visualization techniques for hierarchically structured data like icicle maps. Another common way to visualize a hierarchical dataset is treemaps. Building treemaps involve a usage of tiling algorithms. One of them is widely used and introduced in Lecture 7, namely Slice and Dice algorithm. Before implementation, you have to understand how the algorithm work on a simple data structure.

Word Clouds

Text visualizations became more popular with the growing amounts of digitized, textual data. These kinds of visualizations explain relations between textual entities, abstract and summarize text passages and aim for giving users information about the text to help them find relevant text passages more quickly.

A Word Cloud is a nice, yet imprecise way to have a look at the textual contents of our data. To get an overview of the movies in our movie database, we will create one Word Cloud per genre based on the movie descriptions.

In a first step, you will implement the word cloud using the library d3-cloud. An example of how to use it can be found on D3-graph Gallery. As always, you are not allowed to copy the function, but you can take inspiration from it and adapt it to your needs. The library is already included in the wordcloud.js.

Task 4: Word Cloud Implementation (40%)

Implement the missing parts of the word cloud implementation in the wordcloud.js. Your implementation will automatically be feed with the movie data given in the index.js. The selection drop down menu is already implemented for you. It contains all the genres/group of movies in the dataset. The word cloud should be updated when the selection is changed.

Hints:

  • Define the layout variable using the cloud function from the d3-cloudlibrary.
  • The update function is already defined for you. It is responsible to update the word cloud layout at the start and when the genre selection is changed. Hence, you need to change the words inside the layout with the new 100 most frequent words of the selected genre/group.
  • The draw function is also already defined for you. It is responsible to draw the word cloud layout. It takes an array-like parameter words which is passed implicitly when the draw function is called on the end of the layout at which new list of 100 most frequent words should already be updated.
  • Calling the draw function at the end of the layout will ensure that the word cloud is drawn after the layout is finished.
  • To implement the draw function, you need to create a text element for every word in the words parameter. Since the update is called every time the genre selection is changed, you need to remove the previous text elements before creating new ones. To do so, you have to use join function instead of enter and append functions.
  • The font size you are using for the word cloud depends on the frequency of the word. The more a word is frequent, the bigger the font size should be. The frequency is already calculated for you using wordsPerGroup helping function and can be accessed using the size attribute of the word object.
